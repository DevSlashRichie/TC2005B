Preguntas y Respuestas sobre Tecnologías Web

¿Cómo se puede aplicar CSS a un documento HTML?

Existen tres formas principales de aplicar CSS:

  1. CSS Externo: Mediante un archivo separado .css enlazado con la etiqueta <link> en el head. Es la mejor práctica para mantenibilidad.
  2. CSS Interno: Utilizando la etiqueta <style> dentro del head del documento HTML.
  3. Estilos en línea (Inline): Mediante el atributo style directamente en las etiquetas HTML. No se recomienda para proyectos grandes.

¿Qué es un selector CSS?

Un selector es un patrón de elementos que le indica al navegador qué elementos HTML deben ser seleccionados para aplicarles las reglas de estilo (declaraciones) definidas dentro de las llaves.

¿Cuál es la diferencia entre un ID y una Clase?

La principal diferencia radica en la unicidad y reutilización:

  • ID (#id): Debe ser único en todo el documento. Se usa para identificar un solo elemento específico.
  • Clase (.clase): Puede aplicarse a múltiples elementos en el mismo documento. Se usa para aplicar estilos comunes a varios elementos.

¿Qué es la especificidad en CSS?

Es el conjunto de reglas que utiliza el navegador para decidir qué declaración de estilo se aplica a un elemento cuando hay múltiples reglas que compiten. Se calcula basándose en el tipo de selectores (ID tiene más peso que Clase, y Clase más que Elemento).

¿Qué es el "Box Model" o Modelo de Caja?

Es la base del diseño web en CSS. Cada elemento se representa como una caja rectangular que consta de cuatro partes:

  • Content: El contenido real (texto, imágenes).
  • Padding: El espacio entre el contenido y el borde.
  • Border: La línea que rodea el padding y el contenido.
  • Margin: El espacio exterior al borde que separa al elemento de otros.

¿Cuál es la diferencia entre Internet y la World Wide Web?

Según GeeksforGeeks: "El Internet es una red global de redes interconectadas que permite que dispositivos de todo el mundo se comuniquen. Está compuesto por hardware (routers, cables, satélites, servidores y computadoras), protocolos (TCP/IP) y servicios (correo electrónico, FTP, VoIP)."

Mientras que "la World Wide Web (WWW) es un sistema de páginas web y sitios web conectados a través de hipervínculos y accesibles mediante Internet. Fue inventada en 1989 por Tim Berners-Lee en el CERN."

La diferencia fundamental es que Internet es la infraestructura (las carreteras), mientras que la Web es un servicio que corre sobre esa infraestructura (el tráfico que circula por las carreteras).

¿Cuáles son las partes de un URL?

Una URL (Uniform Resource Locator) está compuesta por las siguientes partes:

  1. Protocolo: Indica el método de acceso (http, https, ftp, etc.)
  2. Subdominio: Parte opcional que precede al dominio principal (www, blog, etc.)
  3. Dominio: Nombre del servidor donde se aloja el recurso
  4. Puerto: Número de puerto opcional (:80 para HTTP, :443 para HTTPS)
  5. Ruta: Ubicación específica del recurso en el servidor
  6. Parámetros de consulta: Datos adicionales que comienzan con ?
  7. Fragmento: Identificador de sección específica precedido por #

Ejemplo: https://www.ejemplo.com:8080/ruta/pagina?id=123#seccion

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Según la documentación de MDN Web Docs, los métodos HTTP son verbos que indican el propósito de la solicitud:

GET
Solicita una representación del recurso especificado. Las solicitudes GET solo deben recuperar datos y no deben contener contenido en el cuerpo de la solicitud. Es seguro, idempotente y cacheable.
HEAD
Solicita una respuesta idéntica a GET pero sin el cuerpo de respuesta. Útil para obtener metadatos sin transferir todo el contenido.
POST
Envía una entidad al recurso especificado, causando un cambio de estado o efectos secundarios en el servidor. Se usa para crear recursos nuevos.
PUT
Reemplaza todas las representaciones actuales del recurso de destino con el contenido de la solicitud. Es idempotente.
PATCH
Aplica modificaciones parciales a un recurso. A diferencia de PUT que reemplaza todo el recurso, PATCH solo actualiza los campos especificados.
DELETE
Elimina el recurso especificado. Es idempotente.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML con usuario y contraseña? ¿Por qué?

Se debe utilizar el método POST.

Según MDN Web Docs: "El método POST envía una entidad al recurso especificado, a menudo causando un cambio de estado o efectos secundarios en el servidor."

Las razones son:

  • Los datos se envían en el cuerpo de la solicitud, no en la URL, lo que proporciona mayor seguridad
  • Los datos sensibles (contraseñas) no quedan visibles en el historial del navegador ni en los logs del servidor
  • POST está diseñado para operaciones que modifican el estado del servidor (como autenticar un usuario)
  • Los datos tienen mayor capacidad de almacenamiento que en GET

¿Qué método HTTP se utiliza cuando se accede a una página a través de un URL?

Se utiliza el método GET.

Según MDN Web Docs: "El método GET solicita una representación del recurso especificado. Las solicitudes usando GET solo deben recuperar datos."

Cuando escribes una URL en el navegador o haces clic en un enlace, el navegador envía automáticamente una solicitud GET para obtener el contenido de esa página.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

El código 200 OK significa que la solicitud tuvo éxito. No ocurrió ningún error.

¿Es responsabilidad del desarrollador corregir un error 404? ¿Por qué?

Sí, es responsabilidad del desarrollador o administrador del sitio web.

Según MDN Web Docs: "El código 404 Not Found significa que el servidor no puede encontrar el recurso solicitado. En el navegador, esto significa que la URL no es reconocida."

El error 404 ocurre (algunos ejemplos) cuando:

  • Una página ha sido eliminada o movida
  • La URL está mal escrita

El desarrollador debe corregirlo porque puede deberse a enlaces rotos internos, páginas eliminadas sin redirección adecuada, o problemas en la estructura del sitio que afectan la experiencia del usuario y el SEO.

¿Es responsabilidad del desarrollador corregir un error 500? ¿Por qué?

Sí, definitivamente es responsabilidad del desarrollador.

Según MDN Web Docs: "El servidor ha encontrado una situación que no sabe cómo manejar. Este error es genérico, indicando que el servidor no puede encontrar un código de estado 5XX más apropiado para responder."

El error 500 (Internal Server Error) indica un problema en el lado del servidor como:

  • Errores en el código de la aplicación
  • Problemas de configuración del servidor
  • Errores en la base de datos
  • Fallos en scripts del lado del servidor

El desarrollador debe identificar y corregir la causa raíz, ya que este error indica un fallo en la lógica del servidor.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado? Menciona elementos HTML 4 desaprobados en HTML5

Un elemento o atributo deprecated (desaprobado) significa que aunque todavía funciona, ya no se recomienda su uso porque puede ser eliminado en versiones futuras o hay alternativas mejores disponibles.

Según MDN Web Docs, elementos de HTML 4 que están desaprobados en HTML5 incluyen:

  • <center> - Usar CSS text-align: center
  • <font> - Usar CSS font-family, font-size, color
  • <b> - Usar <strong> para importancia semántica
  • <i> - Usar <em> para énfasis semántico
  • <strike> o <s> - Usar CSS text-decoration: line-through
  • <u> - Usar CSS text-decoration: underline
  • <frameset> y <frame> - Reemplazados por iframes
  • <big>, <tt>, <marquee>, <blink>

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Comparación entre HTML 4 y HTML5
Característica HTML 4 HTML5
Doctype Complejo y largo Simplificado: <!DOCTYPE html>
Elementos semánticos No existen (usaba div con clases) Nuevos: header, nav, article, section, aside, footer, main
Multimedia Requiere plugins (Flash) Elementos nativos audio y video
Gráficos No soportado nativamente Elemento canvas para gráficos 2D/3D
Formularios Tipos básicos de input Nuevos tipos (ejemplos): email, url, number, date, range, etc.
Almacenamiento local No disponible localStorage y sessionStorage
APIs JavaScript Límitadas Geolocation, Web Workers, WebSockets, etc.
Validación DTD externo requerido Sintaxis más flexible y validación simplificada

¿Qué componentes de estructura y estilo tiene una tabla?

Según MDN Web Docs, una tabla HTML tiene los siguientes componentes de estructura:

  • <table> - Contenedor principal de la tabla
  • <caption> - Título o leyenda de la tabla
  • <thead> - Agrupa las filas del encabezado
  • <tbody> - Agrupa las filas del cuerpo de datos
  • <tfoot> - Agrupa las filas del pie de tabla
  • <tr> - Define una fila de celdas
  • <th> - Celda de encabezado (header cell)
  • <td> - Celda de datos estándar (data cell)
  • <colgroup> - Define un grupo de columnas
  • <col> - Define atributos para una o más columnas

Atributos de estilo (en HTML5 se recomienda usar CSS):

  • border - Define el borde de la tabla y celdas
  • colspan - Celda que abarca múltiples columnas
  • rowspan - Celda que abarca múltiples filas
  • scope - Define si un encabezado aplica a fila, columna o grupo

¿Cuál es el ciclo de vida de los sistemas de información?

El ciclo de vida de los sistemas de información (SI) comprende las etapas por las que pasa un sistema desde su concepción hasta su retiro:

  1. Planificación: Identificación de necesidades y viabilidad del proyecto
  2. Análisis: Estudio detallado de requisitos y procesos actuales
  3. Diseño: Arquitectura del sistema, interfaces y bases de datos
  4. Desarrollo/Implementación: Codificación y construcción del sistema
  5. Pruebas: Verificación de funcionamiento correcto
  6. Instalación/Despliegue: Puesta en producción del sistema
  7. Uso y mantenimiento: Operación continua y corrección de errores
  8. Retiro: Sustitución o eliminación del sistema

¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de desarrollo de sistemas (SDLC: Systems Development Life Cycle) es un proceso estructurado para crear sistemas de información:

Metodologías tradicionales:

  • Cascada (Waterfall): Etapas secuenciales sin retroceso
  • Espiral: Desarrollo iterativo con análisis de riesgos
  • Prototipos: Creación rápida de versiones preliminares

Metodologías ágiles modernas:

  • Scrum: Sprints iterativos de 2-4 semanas
  • Kanban: Flujo continuo de trabajo visualizado
  • XP (Extreme Programming): Énfasis en prácticas técnicas

Las fases comunes incluyen las que ya mencionamos.
